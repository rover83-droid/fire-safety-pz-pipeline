# Миграция `mpb-pz-flow` В Claude Code

Эта папка подготовлена для переноса в Claude Code.

## Что Копировать

Скопируйте архив:

```text
mpb_pz_flow_claude_code_migration.zip
```

или всю рабочую папку проекта, кроме служебного мусора:

```text
logs/
.test-runs/
.test-tmp/
__pycache__/
*.pyc
Новая папка/
```

Папка `Новая папка/` является дублем/старой копией и не нужна для миграции.

## Что Важно Внутри

```text
CLAUDE.md
SKILL.md
agents/
docs/
schemas/
src/
tests/
projects/
web/
pyproject.toml
run_gui.py
эталон МПБ Ф5.1.md
```

## Первый Запуск В Claude Code

Откройте перенесенную папку в Claude Code и напишите:

```text
Прочитай CLAUDE.md, SKILL.md и agents/orchestrator.md. 
Работай как оркестратор ПЗ МПБ. 
Покажи статус проекта demo.
```

Для реального проекта:

```text
Прочитай CLAUDE.md. Продолжить проект hvo. Покажи статус.
```

## Проверка После Переноса

В терминале Claude Code:

```powershell
$env:PYTHONPATH='src'
python -m unittest discover -s tests -v
python -m mpb_pz_flow.cli codex-status --project-dir .\projects\demo
```

Ожидаемо:

```text
tests OK
Стадия: docx_ready
```

Могут быть warning по demo-нормам, если учебные `source_file` отсутствуют. Это не блокирует пайплайн.

## Как Работать После Миграции

Пишите Claude Code обычными фразами:

```text
Начать новый проект ПЗ МПБ для производственного здания Ф5.1
Покажи статус
Что не хватает для запуска агента 1?
Запусти агента 1
Запусти агента 2
Запусти аудит
Собери финальный DOCX
```

Нормативные документы для конкретного проекта кладите сюда:

```text
projects/<имя-проекта>/standards/
```

## Главное Правило Качества

Каждое проектное решение в пояснительной записке должно иметь видимую ссылку на конкретное требование:

```text
СП 8.13130.2020, п. 5.2, табл. 2
```

или:

```text
ФЗ-123, ст. 68, ч. 4
```

Внутренняя ссылка `{norm:<id>}` нужна только для машинной трассировки и не заменяет видимую нормативную ссылку.
