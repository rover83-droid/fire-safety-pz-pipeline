# Contributing to mpb-pz-flow

Спасибо за интерес к проекту! Этот документ — короткое руководство для тех, кто хочет помочь с кодом, документацией или нормативными данными.

## Сообщить об ошибке или предложить улучшение

Откройте [Issue](../../issues/new/choose) и выберите подходящий шаблон. Чем конкретнее описание (что вы делали, что ожидали, что получили — лучше с текстом ошибки или фрагментом артефакта), тем быстрее можно помочь.

## Запуск проекта локально

```powershell
git clone <url> mpb-pz-flow
cd mpb-pz-flow
python -m venv .venv
.venv\Scripts\activate
pip install -e .[docx]
$env:PYTHONPATH = 'src'
python -m unittest discover -s tests -v
```

Требуется Python 3.11+. Для DOCX-экспорта нужен пакет `python-docx` (устанавливается через `pip install -e .[docx]`).

## Правила для Pull Request

1. Один PR — одна логическая задача. Не смешивайте рефакторинг с фичей.
2. Все тесты должны проходить: `python -m unittest discover -s tests`.
3. Если меняется поведение валидаторов, конвейера или схем артефактов — добавьте тест, воспроизводящий новое поведение.
4. Не нарушайте инвариант целостности: `audit_report.json` производится только конвейером (`cli audit`), агенты пишут находки только в `agent_findings.json`. См. [CLAUDE.md](CLAUDE.md), раздел «Integrity Contract».
5. Не добавляйте новые функции в браузерный GUI (`gui_server.py`, `web/`) — он официально заморожен (deprecated). Основной интерфейс — диалог Claude Code/Codex и CLI.
6. Если меняется поведение, видимое пользователю-инженеру, обновите соответствующий `docs/phase-*.md` или `README.md`.

## Главный принцип проекта

**Система не имеет права придумывать нормативный текст, инженерные значения или категории.** Любое изменение, которое ослабляет проверку цитат (`corpus.quote_matches`), провенанс чисел (`validate_numbers`) или привязку аудита к хэшам артефактов, будет отклонено, если оно не сопровождается равноценной заменой проверки.

Если не хватает данных для конкретного объекта — это фиксируется как «лакуна» (`decisions.lacunae`), а не заполняется предположением. Поведение этого механизма не должно регрессировать.

## Структура кода

- `src/mpb_pz_flow/` — основной пакет (models, validators, pipeline, corpus, calculators, triggers, style, docx_export, front_matter, cli).
- `tests/test_pipeline.py` — тесты, сгруппированные по фазам плана улучшений.
- `agents/*.md` — поведенческие контракты для подагентов (Intelligence, Assembly, Audit).
- `docs/phase-*.md` — описание каждой фазы плана улучшений и причин решений.
- `schemas/*.schema.json` — JSON Schema артефактов.
