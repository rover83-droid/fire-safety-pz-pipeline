# Фаза 5 — экспорт: DOCX по ГОСТ и автогенерация обвязки

Завершающая фаза плана. Модули: `docx_export.py` (переработан), `front_matter.py`.

## 1. DOCX по ГОСТ Р 21.101 / ГОСТ 2.105 (`cli export-docx`)

- гарнитура Times New Roman, основной текст 12 pt, межстрочный 1,15;
- поля: левое 25 мм (подшивка), правое 10 мм, верхнее/нижнее 20 мм;
- выравнивание по ширине, абзацный отступ 12,5 мм (в таблицах снят);
- заголовки чёрные, полужирные, 14/13 pt (без синего Word-дефолта);
- подписи таблиц «Таблица N — Наименование» (имя — из ближайшего заголовка);
- титульный лист (объект, «МЕРОПРИЯТИЯ ПО ОБЕСПЕЧЕНИЮ ПОЖАРНОЙ БЕЗОПАСНОСТИ»,
  Раздел 9 ПД, раздел, год) — отключается `--no-title-page`;
- номер страницы в нижнем колонтитуле (поле PAGE).

Экспорт по-прежнему требует действительной хэш-привязки аудита.

## 2. Автогенерация обвязки (`cli front-matter`, флаг `--front-matter` экспорта)

**Перечень сокращений** — собирается по фактическому тексту против словаря
`standards.ABBREVIATIONS`: в перечень попадает только употреблённое.
Токены-кандидаты без расшифровки CLI показывает отдельно («дополните словарь»).

**Перечень нормативно-правовых актов и нормативных документов** — строится из
`decisions.standard_editions` (единственный источник принятых редакций)
с наименованием и статусом применения (обязательный/добровольный) из манифеста
корпуса и отметкой, зарегистрирован ли источник. Один источник истины —
перечень НД не может разойтись с тем, что реально цитируется.

```powershell
python -m mpb_pz_flow.cli front-matter --project-dir .\projects\hvo
python -m mpb_pz_flow.cli export-docx --project-dir .\projects\demo --front-matter
```
