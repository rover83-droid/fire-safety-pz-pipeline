# Фаза 0 — контур целостности

Цель: вердикт «КОНВЕЙЕР ЗАВЕРШЁН» невозможно получить в обход механических проверок.

## 1. Привязка аудита к артефактам

`audit_report.json` создаёт только `python -m mpb_pz_flow.cli audit`. Отчёт содержит:

- `engine: "mpb-pz-flow.audit_project"` — подпись конвейера; отчёт без неё недействителен;
- `artifact_hashes` — SHA-256 паспорта, журнала решений, норм, матрицы, черновика и находок агента на момент аудита.

`refresh_stage`, `finalize_markdown` и `export_docx` сверяют хэши с текущими файлами
(`pipeline.audit_binding_issues`). Любое расхождение — стадия откатывается на
`draft_ready`, финализация и экспорт блокируются. Рукописный отчёт с `passed: true`
не имеет силы.

## 2. Секционные наборы артефактов

Один раздел ПЗ — один набор: `artifacts/sections/<slug>/{norms.jsonl,
applicability_matrix.jsonl, draft.md, agent_findings.json, audit_report.json,
final.md, final.docx}`. Реестр секций — `artifacts/sections/index.json`
(`io.register_section`). Паспорт и журнал решений — общие для проекта
(`artifacts/passport.json`, `artifacts/decisions.json`): один объект — один
журнал редакций.

Корень `artifacts/` обслуживает только активную секцию из `state.json`
(legacy-раскладка). Префиксные файлы вида `<секция>_draft.md` запрещены —
они не видны валидаторам.

Все валидаторы и команды принимают `--section "<название раздела>"`.

## 3. Канал находок аудит-агента

Агент 3 не пишет отчёт. Его выход — `agent_findings.json` в наборе секции
(схема: `schemas/agent_findings.schema.json`):

```json
{
  "level_1": [{"code": "...", "message": "...", "severity": "error"}],
  "level_2": []
}
```

`cli audit` сливает находки агента с механическими проверками; `severity: error`
блокирует pass, нечитаемый или некорректный файл — сам по себе блокирующая
находка `agent_findings.invalid`. Хэш файла находок входит в привязку отчёта.

## 4. Гейт

```powershell
python -m mpb_pz_flow.cli gate --project-dir <dir> [--section "<раздел>"]
```

Выход 0 — и только он — даёт право объявить «КОНВЕЙЕР ЗАВЕРШЁН». Гейт проверяет:
вердикт аудита действителен (подпись + хэши) и `validate_all` не находит ошибок.

Связанные команды: `sections` (список секций со стадиями), `validate --strict`
(падает и на warning).

## Что НЕ входит в Фазу 0

Верификация цитат по корпусу источников, контроль редакций по журналу решений,
вычислимые триггеры и калькуляторы — это Фазы 1–2 (см. план улучшений).
