# Фаза 2 — вычислительный инженерный слой

Цель: применимость норм и числа в тексте ПЗ перестают быть мнением агента —
они вычисляются из паспорта и имеют проверяемый провенанс.

## 1. Машинно-вычислимые триггеры (`triggers.py`)

Норма может нести структурные условия применимости:

```json
"triggers": [{"param": "height_m", "op": ">=", "value": 10, "unit": "м"}]
```

Операторы: `>=`, `>`, `<=`, `<`, `==`, `!=`, `in`, `contains`, `exists`.
Семантика: все условия истинны → «применимо»; любое ложно → «неприменимо»;
параметр не подтверждён в паспорте → «требует инженерной проверки».

- `cli build-matrix` заполняет матрицу предложениями движка (с объяснением
  вычисления в `passport_basis`).
- `validate_triggers`: расхождение статуса строки с предложением движка —
  блокирующая ошибка `matrix.trigger_mismatch`; с заполненным
  `override_justification` — предупреждение `matrix.trigger_override`
  (переопределение разрешено, но записано).

## 2. Калькуляторы (`calculators.py`, `cli calc-run` / `calc-list`)

Нормативные таблицы закодированы как данные; каждый результат несёт значение,
единицу, формулу с подстановкой и основание (документ, год, пункт/таблица).
За пределами закодированного диапазона — отказ (`CalcError`), не экстраполяция.

| calc_id | Что считает | Основание |
|---|---|---|
| sp8_t3_external_flow_f5 | Расход НПВ производственного здания (категория × степень ОО × объём) | СП 8.13130.2020, табл. 3 |
| sp8_t2_external_flow_public | Расход НПВ общественного здания (подмножество: ≤2 этажа) | СП 8.13130.2020, табл. 2 |

| sp4_8_8_drive_distance | Расстояние проезд—стена (5–8 / 8–10 м) | СП 4.13130.2013, п. 8.8 |

`cli calc-run` пишет реестр `artifacts/calculations.json` (подпись engine,
входит в хэш-привязку аудита). Сборщик добавляет в черновик таблицу
«Расчетные показатели». Табличные данные калькуляторов подлежат сверке с
зарегистрированными источниками корпуса (Фаза 1) — это отмечено в notes
каждого результата.

## 3. Числовая согласованность (`validate_numbers`)

Каждое число с единицей измерения в абзацах черновика обязано встречаться
хотя бы в одном источнике провенанса: паспорт, цитаты/пункты норм секции,
строки матрицы, реестр расчётов, журнал решений. Иначе — блокирующая ошибка
`draft.number_unverified`. Голые числа без единиц (номера пунктов, годы)
не проверяются — их покрывает верификация цитат и видимых ссылок.

При финализации (`final.md`) десятичная точка перед единицей измерения
заменяется на запятую («6.8 м» → «6,8 м»); номера пунктов не затрагиваются.

## 4. Включение в конвейер

- стадия `matrix_ready` требует чистых `validate_matrix` + `validate_triggers`;
- стадия `draft_ready` требует чистых `validate_draft` + `validate_numbers`;
- Уровень 1 аудита: + `validate_triggers`; Уровень 2: + `validate_numbers`;
- `calculations.json` включён в хэш-привязку: правка реестра после аудита
  инвалидирует вердикт.

## Результат на реальных данных (проект hvo)

`calc-run` по паспорту ХВО (V = 32 678,8 м³, кат. В предв., III ст., С0,
78×36 м, h = 12,18 м) выдал 8 расчётов, в т.ч.:

- **расход НПВ 25 л/с** (табл. 3 СП 8.13130.2020, диапазон 20–50 тыс. м³) —
  значение, которого в старом черновике не было вовсе («определяется по
  таблицам СП 8.13130»);
- подъезд с двух сторон (36 м > 18 м), проезд ≥ 3,5 м, расстояние 5–8 м;
- выходы на кровлю требуются (12,18 ≥ 10 м), пожарных лестниц — 2
  (P = 228 м / 200);
- порог риска 1·10⁻⁴ (ст. 93 ФЗ-123).
