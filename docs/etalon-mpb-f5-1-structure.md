# Эталонная структура тома 9 МПБ для Ф5.1

Источник: `эталон МПБ Ф5.1.md`.

Эта структура применяется как канон для проектов Ф5.1. Агент 2 обязан собирать разделы в указанном порядке, а Агент 3 обязан проверять наличие и порядок подразделов.

## Предварительная часть

1. Перечень сокращений.
2. Перечень нормативно-правовых актов и нормативных документов по пожарной безопасности.
3. Термины и определения.

## Основная часть

1. Основные положения.
2. Концепция обеспечения пожарной безопасности.
3. Краткая характеристика объекта.
4. Проектные решения по обеспечению пожарной безопасности генеральной планировки.
5. Проектные решения по наружному противопожарному водоснабжению.
6. Проектные решения по обеспечению огнестойкости, конструктивным и объемно-планировочным решениям.
7. Проектные решения по обеспечению безопасности людей при возникновении пожара.
8. Проектные решения по обеспечению безопасности подразделений пожарной охраны при ликвидации пожара.
9. Сведения о категории здания и помещений по признаку взрывопожарной и пожарной опасности.
10. Инженерные системы противопожарной защиты.
11. Проектные решения по системам электроснабжения и молниезащиты.
12. Организационно-технические мероприятия.
13. Выводы.
14. Графическая часть.

## Обязательные вложенные подразделы

### Раздел 6

- 6.1 Степень огнестойкости и конструктивной пожарной опасности.
- 6.2 Деление на пожарные отсеки.
- 6.3 Предел огнестойкости и пожарная опасность строительных конструкций.
- 6.4 Предотвращение распространения пожара.

### Раздел 7

- 7.1 Общие проектные решения.
- 7.2 Эвакуационные выходы.
- 7.3 Эвакуационные пути.
- 7.4 Проектные решения для отдельных зданий и частей здания.

### Раздел 10

- 10.1 Система пожарной автоматики, взаимодействие и управление инженерными системами.
- 10.2 Система пожарной сигнализации.
- 10.3 Система оповещения и управления эвакуацией людей при пожаре.
- 10.4 Аварийное эвакуационное освещение.
- 10.5 Автоматическая установка пожаротушения.
- 10.6 Система удаления продуктов горения при пожаре.

## Правила для агентов

- Не заменять эту структуру общим перечнем разделов ПП РФ N 87.
- Для Ф5.1 использовать именно этот порядок подразделов.
- Если данных для подраздела нет, подраздел не удалять: фиксировать нормативную или исходную лакуну.
- Текст каждого подраздела должен опираться на `norms.jsonl` и `applicability_matrix.jsonl`.
- Аудит должен проверять не только нормативные ссылки, но и соответствие структуре эталона.
